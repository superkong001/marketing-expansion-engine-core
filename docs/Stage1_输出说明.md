# Stage1 输出说明

Stage1 差异分析会生成两类 CSV：**连续型特征差异**（`numeric_diff_*.csv`）与**离散型特征差异**（`categorical_diff_*.csv`）。输出表头为**业务可读列名**（中文），无需数据字典即可理解。本文档说明各列含义及如何阅读结果。

---

## 一、numeric_diff_*.csv（连续型特征差异）

**含义**：全量客群与对比客群在**连续型指标**上的统计差异，以及是否给出阈值推荐。

### 列说明（当前导出为业务可读表头）

| 列名 | 含义 |
|------|------|
| 字段ID | 字段 ID（如 GPRS_MISS_FLOW、CALL_NUM） |
| 字段名称 | 字段中文名（如 总流量、通话次数） |
| 统计月份 | 统计月份（如 202512） |
| 全量均值 | 全量客群该指标的**均值** |
| 对比客群均值 | 对比客群该指标的**均值** |
| 均值差异（对比−全量） | 差异 = 对比−全量（正表示对比更高，负表示对比更低） |
| 相对差异 | 相对差异比例（如 -1.0 表示对比相对全量降为 0） |
| 效应量 | 效应量（Cohen's d），衡量差异幅度；\|效应量\| ≥ 0.2 通常视为“有差异” |
| 中位数差异 | 中位数差异（对比 − 全量） |
| P95分位数差异 | P95 分位数差异 |
| 四分位距差异 | 四分位距（IQR）差异 |
| 变异系数差异 | 变异系数（CV）差异 |
| 综合差异分数 | 综合差异分数（越大表示差异越明显），用于排序 |
| 是否显著差异 | 是否显著差异（True/False，基于效应量） |
| 分布类型 | 基于分位数规则检测：**zero_inflated**（P50=0 且 P90>0）/ **heavy_tail** / **skewed** / **symmetric** |
| 近似偏度 | 近似偏度：(P75+P25−2×P50)/IQR |
| 尾部厚度 | 尾部厚度：(P99−P95)/(P95−P50)，分母≤0 时置 0 |
| 变异系数 | 全量该指标的变异系数 |
| 推荐区间下界 / 推荐区间上界 | 推荐阈值区间下界/上界（空表示未给出推荐） |
| 推荐方向 | 推荐方向：high=高值侧规则，low=低值侧规则 |
| 对比客群估算覆盖率 / 全量估算覆盖率 / 估算Lift | 在推荐区间下的估算覆盖率、Lift；**仅作 backward-compatible 与 debug**，不参与阈值决策（空表示未算） |
| 规则描述 | 规则描述（如“字段 ∈ [a, b]”或“字段 > 0”） |
| 推荐理由 | 推荐理由（如 “symmetric: P25_full, P75_full”、“heavy_tail: P95_full (fallback …)”、“zero_inflated: >0”） |
| 是否给出阈值推荐 | 是否成功给出阈值推荐（True/False） |

### 阅读建议

- 按 **综合差异分数** 从大到小看，越靠前的字段“全量 vs 对比”差异越大。
- 若 **对比客群均值** 接近 0 而 **全量均值** 较大，说明对比客群在该指标上整体很低，差异极大。
- **是否给出阈值推荐** 为 False 时，**推荐理由** 会说明原因（如 “symmetric: no suitable threshold found”），**推荐区间下界/上界** 为空属正常。阈值推荐按分布类型固定策略给出，不再依赖覆盖率/Lift 条件。

---

## 二、categorical_diff_*.csv（离散型特征差异）

**含义**：全量客群与对比客群在**离散型指标**（枚举/分类）上的分布差异，以及推荐的类别集合。

### 列说明（当前导出为业务可读表头）

| 列名 | 含义 |
|------|------|
| 字段ID | 字段 ID（如 MGYY_PREFER、TB_PREFER） |
| 字段名称 | 字段中文名（如 咪咕音乐偏好等级、淘宝偏好等级） |
| 统计月份 | 统计月份（如 202512） |
| 占比差异绝对值之和 | 各枚举值“占比差异”的绝对值之和（越大表示分布差异越大） |
| 最大单类占比差异 | 单个枚举值上最大的占比差异 |
| 差异最大类别及差值 | 差异最大的若干枚举值及差值，如 `nan(+0.9068); 0.0(-0.8970)` 表示：nan 在对比里占比高约 0.91，0.0 在对比里低约 0.90 |
| 推荐类别（圈定>全量） | 推荐“圈定>全量”的类别（用于规则），逗号分隔；nan 表示缺失值被算进推荐 |
| 熵差异 | 熵的差异（对比 − 全量），反映分布集中/分散程度变化 |
| 基尼系数差异 | Gini 系数的差异（对比 − 全量） |
| 综合差异分数 | 综合差异分数（用于排序） |
| 推荐类别集合 | 最终推荐的类别集合（用于规则，如 “字段 in {A,B,C}”） |
| 对比客群命中占比 / 全量命中占比 | 推荐类别在对比/全量上的命中占比 |
| 覆盖增幅 | 对比命中占比/全量命中占比（>1 表示对比更集中在该类） |
| 全量命中人数 / 对比客群命中人数 | 全量/对比中命中推荐类别的人数 |
| 推荐类别个数 | 推荐类别个数 |
| 规则描述 | 规则描述（如 “咪咕音乐偏好等级 in {nan}”） |

### 阅读建议

- 按 **综合差异分数** 从大到小看，越靠前说明该离散字段“全量 vs 对比”的分布差异越大。
- **差异最大类别及差值** 中出现 **nan(+0.9068)** 表示“缺失/未知”在对比客群中占比很高；**推荐类别（圈定>全量）** 为 nan 时，表示算法把“圈定>全量”的类别判成了缺失值。
- **规则描述** 中 “in {nan}” 表示当前推荐的是“缺失”这一类；业务上可保留“缺失”作为一类参与 Stage2 规则组合。

---

## 三、文件在流程中的位置

- **numeric_diff_***：供 Stage2 / 规则引擎使用，用于判断哪些连续指标差异大、分布类型，以及（若有）推荐区间与方向。
- **categorical_diff_***：供 Stage2 / 规则引擎使用，用于判断哪些离散指标差异大、推荐取哪些枚举值作为规则条件。

**总结**：每一行对应一个字段的“全量 vs 对比”差异摘要及可选的阈值/类别推荐；表头为中文业务可读列名，按 **综合差异分数** 排序，优先关注靠前的字段即可。

Stage2 输出说明见：[Stage2 输出说明](Stage2_输出说明.md)。
