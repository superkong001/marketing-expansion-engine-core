# 统一配置说明

Stage1 与 Stage2 共用一份配置文件 `config.json`（位于项目根目录）。修改该文件即可调整参数与功能开关，无需改代码。

## 配置文件位置

- **默认路径**：项目根目录下的 `config.json`（即 `AICohortInsight/config.json`）
- **自定义路径**：运行时可指定
  - Stage1：`python run_stage1.py --config /path/to/config.json ...`
  - Stage2：`python run_stage2.py --config /path/to/config.json ...`

## 配置文件结构

```json
{
  "stage1": { ... },
  "stage2": { ... }
}
```

- **stage1**：阶段1（特征差异计算与阈值推荐）所用参数
- **stage2**：阶段2（原子规则、Beam Search、客群组合）所用参数  
若文件中只有 stage2 的键（无 `stage1`），则整份 JSON 作为 stage2 配置（兼容旧版仅 Stage2 的配置文件）。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `top_k_numeric` | 连续特征阈值推荐 TopK | 50 |
| `top_k_categorical` | 离散特征类别推荐 TopK | 30 |
| `target_ratio` | 正态分布时目标覆盖比例（反推 k），不设则 null | null |
| `distribution.skew_eps` | 分布检测偏度阈值 | 0.2 |
| `distribution.tail_thr` | 分布检测尾部比阈值 | 0.5 |
| `distribution.eps` | 数值保护小量 | 1e-9 |
| `numeric_threshold.min_cohort_coverage` | 连续阈值推荐用（仅 debug） | 0.1 |
| `numeric_threshold.min_lift` | 连续阈值推荐用（仅 debug） | 1.2 |
| `numeric_threshold.output_candidate_family` | 是否输出候选阈值族（供 Stage2 原子使用） | true |
| `numeric_threshold.dedup_overlap_threshold` | 候选族去冗余重叠阈值 | 0.8 |
| `categorical_threshold.output_candidate_family` | 是否输出候选类别族 | true |
| `categorical_threshold.dedup_overlap_threshold` | 候选族去冗余重叠阈值 | 0.8 |
| `categorical_threshold.min_delta` | 离散推荐最小差异 | 0.01 |
| `categorical_threshold.min_cov` | 离散推荐最小覆盖率 | 0.1 |
| `categorical_threshold.min_increment` | 离散推荐边际增量 | 0.01 |
| `diff_numeric.effect_size_significant_threshold` | 效应量显著阈值 | 0.2 |
| `diff_numeric.w_effect_size` / `w_delta_median` / `w_delta_p95` | 连续差异评分权重 | 0.5 / 0.3 / 0.2 |
| `diff_categorical.top_k` | 离散差异前 k 个最大差异类别 | 3 |

## Stage2 配置项

### 规则组合与输出

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `max_features_per_segment` | 每个客群最多字段数 | 3 |
| `top_k_features` | 参与组合的 Top 特征数 | 30 |
| `beam_size` | Beam Search 束宽 | 20 |
| `top_n_candidates` | 候选规则保留数 | 30 |
| `max_segments` | 最终客群组合数量（target 未设时使用） | 5 |
| `target_segment_count` | 预期推荐客群数；null 时使用 max_segments | null |
| `overlap_max` | 客群间最大结构重叠度 | 0.6 |
| `different_field_overlap` | 不同字段重叠度阈值 | 0.3 |
| `min_rules_per_segment` | 每个客群至少规则数（k≥该值）；设为 2 时允许 k=2 或 k=3 | 2 |
| `require_exact_k` | 是否只保留严格 k=max_features_per_segment 的候选；false 时允许 k=2 与 k=3 | false |
| `similarity_threshold` | 多客群去重时结构相似度阈值 | 0.95 |
| `similarity_fallback_thresholds` | 相似度阈值回退列表 | [0.95, 0.9, 0.85] |
| `selection_w_dist` / `selection_w_quality` | 两阶段选择时结构距离权重 / 综合得分权重 | 0.6 / 0.4 |

### 评分权重与阈值

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `min_stability_score` | 最小稳定性得分 | 0.3 |
| `min_stability_score_categorical` | 离散原子规则单独稳定性阈值；null 则用 min_stability_score；设低值可放宽以扩大原子池 | null |
| `min_divergence_score` | 最小差异评分 | 0.1 |
| `beam_w_divergence` / `beam_w_stability` / `beam_w_diversity` | Beam 综合评分权重 | 0.5 / 0.3 / 0.2 |
| `w_effect_size` / `w_delta_mean` / `w_quantile_shift` / `w_significance` | 差异评分权重 | 0.4 / 0.3 / 0.2 / 0.1 |
| `stability_w_std` / `stability_w_kurtosis` / `stability_w_sample` | 稳定性评分权重 | 0.4 / 0.3 / 0.3 |
| `missing_pair_penalty_lambda` | Pair 关联缺失惩罚：score_final = score - λ×missing_pair_ratio | 0.1 |

### 原子规则精度过滤（永不清零 + 自动降级）

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `enable_precision_filter` | 是否启用原子规则精度过滤 | true |
| `min_atomic_rules_for_search` | 过滤后至少保留条数，不足则触发降级 | 30 |
| `expected_cohort_ratio` | 先验 sub 占 ALL 比例（建议由业务配置）；缺失时不做了 precision_proxy 硬过滤 | 0.01 |
| `max_base_cov` | Round1 原子规则 ALL 覆盖率上限（放宽可多留原子，如 0.40～0.45） | 0.42 |
| `fallback_max_base_cov` | 降级后 ALL 覆盖率上限 | 0.5 |
| `min_sub_cov` | 原子规则 sub 覆盖率下限 | 0.02 |
| `min_lift_atomic` | Round1 原子规则 lift 下限（放宽可多留原子，如 1.0～1.1） | 1.05 |
| `fallback_min_lift_atomic` | 降级后 lift 下限 | 1.05 |
| `min_precision_mult` | precision_proxy ≥ 该值 × prior_pi（仅当 expected_cohort_ratio 存在时生效） | 1 |

### 原子规则生成与去重

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `atomic_rules.use_candidate_family_first` | 优先从 Stage1 候选族生成原子 | true |
| `atomic_rules.max_atomic_per_column` | 每列最多原子条数 | 5 |
| `atomic_rules.fallback_to_legacy_rec` | 无候选族时回退到 rec_low/rec_high 等 | true |
| `lift_min` | 原子规则 lift 下限（旧） | 1.5 |
| `min_delta` | 离散推荐最小差异 | 0.02 |
| `max_categories` | 离散原子规则最大类别数 | 5 |
| `min_stability_score_categorical` | 离散原子规则单独稳定性阈值；设低可放宽以扩大原子池 | 见代码默认 |
| `dedup.same_structure_at_candidate` | 候选阶段是否按同特征集去冗余 | true |
| `dedup.same_structure_at_portfolio` | 组合阶段是否按同特征集去冗余 | true |

当原子规则数 < `min_atomic_rules_for_search`（30）时，会先触发精度 Round2 降级；若仍不足，则 Round3 放宽 diff/stability 阈值重新生成原子规则，详见处理逻辑文档。更多字段见 `stage2/stage2_config.py` 中的 `Stage2Config`。

## 命令行与配置优先级

- 若提供了 `--config` 且文件存在，则从该文件读取对应阶段配置。
- Stage1：命令行参数 `--top-k-numeric`、`--top-k-categorical`、`--target-ratio` 会覆盖配置文件中相同项（当前实现为“先读配置再以命令行覆盖”的逻辑在 main 中：先加载 config，再传参给 run_diff_analysis，因此若同时传了命令行参数，以命令行为准）。
- Stage2：仅从配置文件或默认配置读取，无对应命令行覆盖。

## 注意事项

- 配置文件中可包含 `_comment` 等键，程序会忽略未在配置类中声明的键。
- 修改配置后重新运行相应阶段即可生效。
